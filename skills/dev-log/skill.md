# dev-log スキル

`~/.claude/Log/` に保存された前回セッションのログを読み込み、何をしたかを整理して提示する。

## トリガー

- `/dev-log` — 前回のセッションログを表示
- 「前回の作業を確認して」「ログを読んで」「前回何したか教えて」「続きから始めて」

## 手順

### ステップ1: 現在プロジェクトの特定

```bash
PROJECT=$(basename "$(pwd)")
```

### ステップ2: ログファイル検索

プロジェクトの `.claude/Log/` を検索する。

```bash
LOG_DIR=".claude/Log"

ls -t "$LOG_DIR/" 2>/dev/null | head -5
```

### ステップ3: ログ選択・読み込み

- 1件 → 自動選択
- 複数件 → 最新を自動選択（ファイル名のタイムスタンプが日時を示す）
- 0件 → 「前回ログなし」として終了

```bash
LOG_FILE=$(ls -t ".claude/Log/" 2>/dev/null | head -1)
[ -n "$LOG_FILE" ] && cat ".claude/Log/$LOG_FILE"
```

### ステップ4: 要約の提示

読み込んだログから以下を整理して提示：

1. **前回の作業日時** — ファイル名のタイムスタンプから
2. **主な依頼内容** — ユーザーメッセージの要点（最初の依頼を中心に）
3. **変更・参照ファイル** — ログの「変更・参照ファイル」セクション
4. **未完了事項** — 会話末尾の状況から推測される残タスク

### ステップ5: 完了報告

```
前回セッション読み込み完了
  ログ: {ファイル名}
  日時: {日時}

要点:
  {要約2〜3行}

続きの作業を教えてください。
```

## ログがない場合

```
前回のセッションログが見つかりませんでした。
（.claude/Log/ にファイルなし、または dev-log-setup 未実行）

新規セッションとして開始します。
```

## 注意

- `.claude/Log/` は `dev-log-setup` スキルが作成する。未実行なら案内する
- ログは Stop フックが自動生成（セッション終了時）。途中終了では生成されないことがある
