# poc/ — 機能検証（PoC）置き場

仕様確定前に、技術的実現可能性や外部連携を確認するための throwaway コード置き場。
本実装（src/）とは分離する。

---

## フォルダ構成

```
poc/
  {機能名}/
    README.md       # 検証目的・実施方法・結果サマリ
    *.py / *.ts 等  # 検証スクリプト
```

## 検証パターン

| 検証対象 | 方法 |
|---------|------|
| 技術可能性（ライブラリ/SDK/API） | 最小スクリプトで動作確認 |
| 外部連携（API/OAuth/Webhook） | スタブ叩いて疎通確認 |
| 性能（レスポンス時間/スループット） | ベンチスクリプトで実測 |
| データモデル（スキーマでクエリ書けるか） | sqlite 等で SQL 試打 |
| アルゴリズム（計算量・精度） | テストデータで検算 |
| UI/UX | HTML モック or Figma でスクショ |

## PoC 必須／スキップ判断基準

**必須:**
- 新規ライブラリ/SDK 採用
- 外部 API 依存
- 性能要件あり（レスポンス < Nms 等）
- 暗号化・認証など細部が複雑
- 過去経験ない技術領域

**スキップ可:**
- 単純な CRUD
- 標準的な画面遷移のみ
- 既存実装の流用

## 結果の記録

各機能の検証結果は `document/feature/{機能名}.md` の「機能検証ログ」欄に転記する。
PoC コード自体は本実装に流用せず、参考のみ。

## 注意

- PoC は最小コードで書く（エラーハンドリング・テストは省略可）
- 検証完了後は `poc/{機能名}/README.md` に結論を残し、コードは削除せず保管
- 認証情報・本番DB接続は絶対に使わない
