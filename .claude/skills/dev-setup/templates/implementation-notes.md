# implementation-notes

仕様書（`requirements.md` / `spec.md` / `feature/*.md` / `tech-stack.md`）に**書かれていなかった**判断・変更・妥協・意思決定を**実装中に逐次記録**するログ。

---

## 記録ルール

### 記録対象（必ず追記）

- 仕様書に未記載のまま実装段階で決めた事項
- 仕様書と異なる実装にした事項（理由つき）
- 性能・互換性・ライブラリ制約で妥協した事項
- 代替案を検討して却下した事項（却下理由つき）
- 後で見直したい一時的 workaround

### 記録対象外

- 仕様書通りに素直に実装したもの
- 自明なリファクタ・誤字修正

### タイミング

- 判断した**その場**で追記（あとでまとめてはダメ。忘れる）
- コミットと同タイミングで追加するのが理想
- 仕様書の方を更新すべき内容と判断した場合は、仕様書を更新したうえでここに「{日付}: {仕様書名} に反映」と残す

---

## ログ

### YYYY-MM-DD — {タイトル / 対象機能}

- **コンテキスト:** 何をしていて、どこで判断が必要になったか
- **選んだもの:** 実際に採用した実装・設定・ライブラリ
- **理由:** なぜそれを選んだか（性能・コスト・既存資産・期限など）
- **却下した代替案:** 検討したが選ばなかったもの、その理由
- **影響範囲:** どのファイル・どの機能に効くか
- **見直しトリガー:** どういう条件になったら再検討するか（任意）
- **関連:** 関連 Issue / PR / コミット（任意）
